Github flow

Github flow är ett arbetssätt att använda github på. Man skapar branches som man sedan kan uppdatera. Dessa
branches blir åtkomliga via github för dig ifall du skulle använda en annan dator eller dina kollegor du delar
arbetet med. Detta gör att du kan enkelt se och komma åt din och andras kod och hantera olika uppdateringar
och förändringar i koden. När man känner sig klar kan man föreslå sammanslagning av kod. när detta görs kommer
du och dina kollegor kunna se över de förändringar du har gjort.

Flexbox

Flexbox är ett sätt att skapa boxar i CSS. Man skapar boxar som förändars beroende på hur du skriver koden och
andra boxar som tillhör boxen. Man kan skapa boxar i boxar för att skapa en komplexstruktur.
Fördelen med Flexbox är att det blir enklare att få kontroll på ditt innehåll på sidan.

Boxarna läggs brevid varandra beronede på om du vill ha den på bredden(row) eller höjden(colums). Du kan skapa
en row som du sedan lägger in colums i och viceverse. Detta är det som gör att du kan bygga väldigt komplexa sidor
enkelt. När man gör detta hamnar de bara brevid varandra och man måste använda "vanlig" CSS för att få den stil
man är uteefter. Boxarna är som väggarna i ditt hus som du sedan måste själv måla och inreda.

DOM

Dom är en gränssint som ger dig tillgång till html via tex Javascript. Detta gör så att du kan manipulera
HTML när du behöver den. Istället för att ha all HTML liggandes skapar man via koden ny HTML för just det du
behöver.

Fin kod

Att skriva fin kod menas att man skriver kod som är lättläsligg så att så många som möjligt ska förså vad som
händer. Detta gör att koden blir enklar att läsa men även att koda med. Har du "ful" kod är det svårt att få
överblick vad koden ska göra och kan bli svårt att hitta fel och buggar i den egna koden. Man bör namnge
variabler på ett bra sätt så att det är tydligt vad de är till för. Om en variabel bara heter x är det nästan
helt omöjligt att veta vad den gör. Man kanske själv tror man har koll för att sedan gå på fika och komma
tillbaka och då glömt vad x gör. Detta gäller även funktioner och klasser.

Även att försöka ha en struktur i koden. Inte slänga runt funktioner och variabler på måfå i koden. Till exempel
om jag nu helt plötlsigt skulle börja svara på frågan om ramverk för att sedan komma tillbaka till fin kod i
nästa mening skulle det bli väldigt svårt att förstå.

Ramverk

Ramverk är kod som skapats av andra för att förenkla och slippa skriva ny kod varje gång. När man programmerar upptäcker man att visa problem uppstår flera gånger i olika program. Istället för att skriva om samma kod varje gång man stöter på problemet skapar man istället ramverk. Angular är ett ramverk som skapats av google. Det används för att skapa framför allt singlepage appplikationer. NodeJS är ett open source ramverk som används för att skriva javascript kod mot servrar.
